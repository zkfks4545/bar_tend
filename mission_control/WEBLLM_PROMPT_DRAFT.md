# WebLLM 캐릭터 프롬프트 초안

## 상태

이 문서와 `bar_tend/src/lib/webllm/prompts/character-prompts.ts`의 프롬프트는 **초기 예시 초안**이다.

최종 후보 모델을 대상으로 `RST-603-B` RP 평가를 수행하고, `RST-604`에서 실제 WebLLM 대화 흐름에 연결한 뒤 **반드시 수정해야 한다**. 현재 문구를 최종 프롬프트나 확정된 토큰 구성으로 간주하지 않는다.

## 목적

- 카루아와 시에스타의 말투 차이를 LLM이 읽을 수 있는 시스템 프롬프트 형태로 변환한다.
- 캐릭터 표현과 추천 엔진의 책임을 분리한다.
- RST-402가 선택한 질문 주제를 설문처럼 보이지 않는 카루아식 질문으로 표현한다.
- 추천 결과와 추천 상태는 LLM이 임의로 변경하지 못하게 한다.

## 구현된 프롬프트

| 이름 | 역할 |
|---|---|
| `KARUA_SYSTEM_PROMPT` | 카루아 일반 대화 및 표현 원칙 |
| `SIESTA_EVENT_SYSTEM_PROMPT` | 시에스타의 짧은 난입 이벤트 |
| `ENSEMBLE_EVENT_SYSTEM_PROMPT` | 카루아·시에스타 합동 만담 |
| `buildKaruaQuestionPrompt()` | 추천 엔진이 고른 질문 주제를 카루아 말투로 표현 |
| `buildKaruaRecommendationPrompt()` | 확정된 추천 결과를 변경하지 않고 카루아식으로 설명 |
| `buildSignalExtractionPrompt()` | 자유 답변에서 추천 상태 후보만 JSON으로 추출 |
| `getCharacterSystemPrompt()` | RP 평가 대상에 맞는 캐릭터 프롬프트 선택 |

## 책임 경계

- LLM은 대사 표현, 질문 표현, 추천 이유 표현을 담당한다.
- 추천 엔진은 질문 주제, 추천 상태, 칵테일 결과를 결정하고 검증한다.
- 상태 추출 결과는 후보일 뿐이며 추천 엔진의 검증 없이 반영하지 않는다.
- 시에스타는 낮은 빈도의 짧은 이벤트 캐릭터이며 상시 대화 주도권을 갖지 않는다.

## 후속 수정 기준

- `RST-603-B`: 캐릭터별 수동 평가에서 말투 혼합, 장문, 설교, 과도한 철학, 농담 강요를 확인한다.
- `RST-604`: 실제 질문·추천·자유 답변 흐름에서 프롬프트 길이와 상태 전달 형식을 조정한다.
- 최종 모델 확정 후 모델별 지시 준수력에 맞춰 예시 수와 금지 규칙을 줄이거나 보강한다.
- 반복 출력, 예시 문장 복제, 프롬프트 주입, JSON 형식 이탈을 측정하고 보완한다.

## 현재 한계

- 최종 모델에서 실행 검증하지 않았다.
- 토큰 수와 KV 캐시 비용을 최적화하지 않았다.
- 예시 대사를 그대로 반복하는 앵커링 위험을 측정하지 않았다.
- 상태 추출 JSON의 런타임 파서 연결은 RST-604 범위에 남아 있다.
