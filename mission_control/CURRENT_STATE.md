> 최종 갱신일: 2026-06-13 (DEC-018 추천 질문 4축 간소화)

## 상태 요약

| 항목 | 현재 상태 |
|---|---|
| 목표 제품 | Re:Station 카루아 중심 대화형 칵테일 추천 MVP + 시에스타 만담 이벤트 |
| 현재 구현 | Re:Station 브랜드와 카루아 규칙 대화를 적용한 추천 프로토타입 |
| 실제 애플리케이션 경로 | `bar_tend/` |
| 현재 단계 | 적응형 JSON 질문과 선택 버튼 완료, 주요 흐름 테스트 확대 전 |
| 기술 방향 | React + Vite + 프론트엔드 단독, WebLLM 잠정 보류 |
| 추천 원칙 | JSON·DB·규칙 기반으로 전체 답안 확정, WebLLM은 향후 말투 포장만 허용 |
| 빌드 | 통과: `npm run build` (`tsc --noEmit` 포함) |
| 린트 | 통과: `npm.cmd run lint` |
| 자동 테스트 | Vitest 35개 통과 |

## 현재 구현과 목표의 주요 차이

| 영역 | 현재 구현 | 목표 |
|---|---|---|
| 브랜드 | Re:Station 적용 | Re:Station 유지 |
| 캐릭터 | 카루아 규칙 응답과 계약 평가 적용 | 규칙·JSON 대사 확장 |
| 대화 스타일 | 농담 우선, 직접 위로 방지, 안전 의도 우선 처리 | WebLLM 없이 전체 흐름 완성 |
| 추천 모델 | 클래식 29종과 시그니처 2종의 단일 `CocktailData` 컬렉션 | 단일 데이터 모델 |
| 추천 입력 | 적응형 JSON 질문, 선택지 버튼, 자유 입력과 보조·취소 동작으로 상태 수집 | 주요 사용자 흐름 테스트 확대 |
| 추천 출력 | 중립적인 상세 정보 카드, 정확 추천과 최근접 타협안을 구분한 카루아식 대화 멘트 | 동일 |
| 생성형 대화 | 규칙 기반 엔진 | WebLLM은 말투 포장 용도로만 잠정 보류 |
| UI 색상 | 다크브라운/골드 + 보라 언더톤·네온·명암비 (DEC-014) | 동일 (완료) |
| 테스트 | 데이터·검색·카루아 계약·적응형 질문·입력 라우팅 회귀 테스트 35개 | 저장, 주요 흐름 테스트 확대 |
