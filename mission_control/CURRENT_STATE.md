## 상태 요약

| 항목 | 현재 상태 |
|---|---|
| 목표 제품 | Re:Station 카루아 중심 대화형 칵테일 추천 MVP + 시에스타 만담 이벤트 |
| 현재 구현 | Re:Station 브랜드와 카루아 규칙 대화를 적용한 추천 프로토타입 |
| 실제 애플리케이션 경로 | `bar_tend/` |
| 현재 단계 | 데이터 계약 통합 완료, 애플리케이션 로직 분리 착수 전 |
| 기술 방향 | React + Vite + 프론트엔드 단독 + WebLLM |
| 추천 원칙 | DB 및 규칙 기반 결정, WebLLM은 대화와 추천 설명만 담당 |
| 빌드 | 통과: `npm run build` (`tsc --noEmit` 포함) |
| 린트 | 통과: `npm.cmd run lint` |
| 자동 테스트 | Vitest 9개 통과 |

## 현재 구현과 목표의 주요 차이

| 영역 | 현재 구현 | 목표 |
|---|---|---|
| 브랜드 | Re:Station 적용 | Re:Station 유지 |
| 캐릭터 | 카루아 규칙 응답과 계약 평가 적용 | WebLLM 표현 연결 |
| 대화 스타일 | 농담 우선, 직접 위로 방지, 안전 의도 우선 처리 | WebLLM에서도 동일 계약 유지 |
| 추천 모델 | 단일 `CocktailData` 컬렉션 | 단일 데이터 모델 |
| 추천 입력 | 단맛, 신맛, 도수, 탄산 중심 | 기분, 상황, 맛 취향, 무알코올, 제외 재료 |
| 추천 출력 | 카드와 일반 대사 | 구조화된 추천 이유와 카루아식 한마디 포함 |
| 생성형 대화 | 규칙 기반 엔진, 레거시 OpenAI/Ollama 제거 | Web Worker 기반 WebLLM |
| UI 색상 | 갈색 및 골드 중심 | 어두운 남색 또는 흑회색과 연보라 |
| 테스트 | 데이터·검색·카루아 계약 회귀 테스트 9개 | 저장, 주요 흐름 테스트 확대 |
