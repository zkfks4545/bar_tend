# 인수인계

> 최종 갱신일: 2026-06-12 (RST-605-B WebLLM 런타임 생명주기 감사)

## 현재 목표

기존 BarBot 프로토타입을 **Re:Station 카루아 중심 대화형 칵테일 추천 MVP**로 개편한다. 시에스타는 낮은 빈도의 짧은 만담 이벤트로 등장한다.

## 확정된 핵심 계약

- [x] MVP의 주 대화 캐릭터는 카루아이며, 시에스타는 낮은 빈도의 만담 이벤트로만 등장한다.
- [x] 상담이 아니라 농담과 추천을 통한 환기를 제공한다.
- [x] 추천 결과와 근거는 DB 및 추천 엔진이 결정한다.
- [x] WebLLM은 일반 대화와 추천 설명만 생성한다.
- [x] WebLLM은 브라우저의 Web Worker에서 실행한다.
- [x] OpenAI API 및 Ollama 후보에서 WebLLM 기반 생성형 표현 계층으로 전환한다.
- [x] 공개 자료와 설치된 WebLLM 카탈로그를 기준으로 후보 순위를 정한다. `Qwen3.5-2B-q4f16_1-MLC`는 잠정 1순위이며 최종 모델은 미정이다.
- [x] WebLLM 실패 시 규칙 기반 엔진으로 복구한다.
- [x] 백엔드는 MVP 범위에서 사용하지 않는다.
- [x] 캐릭터 대화 생성과 검수는 `CHARACTER_DESIGN.md`를 기준으로 한다.
- [x] WebLLM 도입 전에는 카루아 말투가 포함된 JSON 질문으로 추천 상태를 수집하고, 도입 후에는 같은 계약에서 WebLLM이 질문 표현과 상태 후보 추출만 담당한다.
- [x] 최종 모델 확정 전에는 사용자가 후보 모델을 명시적으로 선택해 로드하며 코드에서 자동 선택하지 않는다.
- [x] 준비된 WebLLM은 일반 대화와 엔진이 확정한 추천 설명에만 사용하고, 안전 입력·추천 질문·생성 실패는 규칙 엔진으로 처리한다.
- [x] 규칙 응답은 약 80~100ms 안에 먼저 표시하고, WebLLM 토큰이 도착하면 같은 메시지를 점진적으로 갱신한다.
- [x] 일반 대화는 최대 64토큰·최근 이력 4개·5초 예산, 추천 설명은 최대 96토큰·대화 이력 제외를 기본으로 한다.
- [x] 입장 전에 AI 모델의 예상 캐시·GPU 메모리 사용을 알리고, AI 입장을 선택한 경우에만 기본 후보 모델을 자동 로드한다.
- [x] 설문 전용 입장이나 모델 미준비·실패 상태에서는 자유 입력을 숨기고 추천 설문과 레시피북만 제공한다.
- [x] 나가기 옆 캐시 삭제 버튼으로 실행 모델을 언로드하고 알려진 후보 모델 캐시를 삭제할 수 있다.
- [x] 취소·언로드 이후 늦게 완료된 모델 로드가 다시 활성화되지 않으며, 로드 실패 Worker를 정리한다.
- [x] 시간 초과·취소는 모델을 재사용하고, 실제 생성 오류는 ERROR 상태로 전환해 반복 실패를 막은 뒤 재로드할 수 있다.

## RST-602-D 신규: 캐릭터 RP 일관성 평가 설계

RP 평가는 공개 벤치마크가 측정하지 못하는 카루아·시에스타의 말투와 관계 대비를 검수한다. 자동 판정은 명백한 금지 표현과 길이 위반만 찾으며, 캐릭터다운지는 사례별 수동 검수 기준으로 판단한다.

- **설계 완료**: RP 사례 8개(카루아 5 + 시에스타 2 + 관계 만담 1), 캐릭터별 수동 검수 기준, 케이스별 자동 금지 패턴
- **중요 원칙**: 사용자가 농담을 멈춰 달라고 하면 카루아는 경계를 존중한다. 농담 고집을 페르소나 유지로 평가하지 않는다.
- **실행 계획**: RST-603-B에서 최종 모델 연결 후 `CHARACTER_DESIGN.md` 기반 프롬프트를 검수한다. 후보 순위 비교 작업은 아니다.
- **참고 파일**: `bar_tend/src/data/karua-evaluation-set.json` (rp-* 항목 8개), `bar_tend/src/lib/evaluation/karua-evaluation.ts`, `mission_control/CHARACTER_DESIGN.md`

## 다중 작업 PC 동기화

이 프로젝트는 여러 PC에서 작업할 수 있다. 히스토리 충돌 시 아래 명령으로 강제 동기화한다.

### 로컬 변경이 없을 때
```bash
git fetch origin
git reset --hard origin/master
```

### 로컬 변경이 있을 때 (변경 보존)
```bash
git stash
git fetch origin
git reset --hard origin/master
git stash pop
```

> 이 명령은 로컬 히스토리를 리모트와 완전히 일치시킨다. `git pull`은 강제 푸시로 인한 분기 히스토리에서 실패할 수 있으므로 `fetch + reset --hard`를 사용한다.

### 규칙
- 이 저장소에서는 `git push --force`를 사용하지 않는다.
- 리모트에 밀어넣기 전에 항상 `git fetch`와 `git rebase`로 최신 상태를 확인한다.

## 주의사항

- [ ] 시에스타를 상시 대화 캐릭터나 선택 캐릭터로 확장하지 않는다.
- [ ] 시에스타 만담 이벤트는 추천 질문, 안전 확인, 퇴장 및 초기화 흐름을 끊지 않는다.
- [ ] 시에스타 이벤트가 끝난 뒤 대화 중심을 카루아에게 돌린다.
- [ ] 시에스타 이벤트는 예고 없는 난입으로 시작하고 창고 정리, 청소, 재고 확인 같은 업무 복귀 발화로 끝낸다.
- [ ] 사용자가 시에스타에게 말을 이어도 시에스타가 상시 대화 캐릭터처럼 남아 있지 않게 한다.
- [ ] 카루아 대사는 농담을 먼저 두고 의미를 직접 해설하지 않는다.
- [ ] WebLLM 연결 전에 규칙 기반 전체 흐름을 완성한다.
- [ ] WebLLM이 추천 결과를 선택하거나 변경하게 만들지 않는다.
- [ ] WebLLM이 추출한 추천 상태 후보를 검증 없이 반영하지 않는다.
- [ ] 최종 모델 결정 전 특정 후보를 제품 기본값으로 고정하지 않는다.
- [ ] 모델 다운로드는 자동 시작하지 않고 사용자에게 크기와 진행 상태를 보여준다.
- [ ] WebLLM을 기다리느라 추천 카드나 규칙 기반 첫 반응 표시를 늦추지 않는다.
- [ ] 준비된 모델 기준 첫 토큰 800ms 목표와 요청 시간 예산을 측정한다.
