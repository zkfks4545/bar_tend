## DEC-013: 단계적 대화형 추천 질문

| 항목 | 내용 |
|---|---|
| 결정 내용 | WebLLM 도입 전에는 질문 주제, 선택지, 상태 반영 규칙을 JSON으로 정의하고 카루아 말투의 질문과 반응 문구를 함께 제공한다. WebLLM 도입 후에는 추천 엔진이 정한 질문 주제를 WebLLM이 자연스럽게 표현하고, 사용자 답변에서 상태 갱신 후보를 추출한다. |
| 공통 계약 | 추천 상태 스키마, 허용 값, 질문 이력, 추천 종료 조건은 WebLLM 도입 전후에 동일하게 유지한다. |
| 추천 엔진 책임 | 다음 질문 주제 선택, 상태 갱신 검증, 후보 필터링, 추천 결과와 근거 결정 |
| WebLLM 책임 | 질문과 반응의 카루아식 표현, 자유 입력에서 구조화된 상태 갱신 후보 추출 |
| 금지 | WebLLM이 추천 상태를 검증 없이 확정하거나 칵테일 후보와 결과를 직접 선택 또는 변경하는 것 |
| 이유 | WebLLM 이전에도 안정적인 추천 흐름을 제공하고, 도입 후 설문 느낌을 줄이면서 추천 재현성과 복구 경로를 유지하기 위해서다. |
| 되돌릴 조건 | 구조화 질문만으로도 충분한 대화 UX를 제공하거나, WebLLM 상태 추출이 목표 기기에서 안정적으로 동작하지 않을 때 |

## DEC-012: 시에스타 만담 이벤트

| 항목 | 내용 |
|---|---|
| 결정 내용 | 카루아를 주 대화 캐릭터로 유지하면서 시에스타가 손님-카루아 대화에 불쑥 난입해 한쪽 또는 양쪽과 짧은 만담을 나눈 뒤 바 업무로 돌아간다. |
| 역할 경계 | 시에스타는 추천 결과를 결정하거나 대화의 주도권을 장시간 가져가지 않는다. 캐릭터 선택 기능도 제공하지 않는다. |
| 기본 길이 | 한 이벤트는 일반적으로 시에스타와 카루아를 합쳐 2~4개의 짧은 발화로 끝낸다. |
| 장면 문법 | 예고 없는 난입으로 시작하고 창고 정리, 청소, 재고 확인 등 일상 업무 복귀로 끝낸다. |
| 발생 조건 | 추천 흐름을 방해하지 않는 일반 대화 구간, 추천 완료 직후, 재방문 또는 충분한 대화 교환 후 |
| 금지 조건 | 안전 확인 중, 사용자가 명시적 질문에 답하는 중, 추천 필터 진행 중, 퇴장 또는 초기화 중 |
| 이유 | 카루아의 주인공성을 유지하면서 두 사람의 관계와 Re:Station의 생활감을 짧게 보여준다. |
| 되돌릴 조건 | 이벤트가 핵심 추천 흐름을 자주 끊거나 시에스타가 카루아보다 강하게 인식될 때 |

## DEC-010: 이름 검색 우선

| 항목 | 내용 |
|---|---|
| 결정 내용 | 입력에 등록된 칵테일 이름 또는 별칭이 포함되면 추천 의도가 함께 있어도 해당 칵테일을 우선 표시한다. |
| 이유 | 사용자가 특정 칵테일을 언급한 경우 일반 취향 탐색보다 의도가 구체적이다. |
| 경계 | 이름 검색은 이름과 별칭만 매칭한다. 재료, 향, 베이스 표현은 취향 기반 추천 흐름으로 전달한다. |
| 예시 | `모히토 추천해줘`는 모히토 표시, `과일 향 추천해줘`는 취향 추천 시작 |
| 되돌릴 조건 | 사용자가 이름을 비교 대상으로 언급하는 부정/제외 의도를 별도 구조화할 때 |

## DEC-011: OpenAI/Ollama에서 WebLLM으로 전환 및 체감 지연 최소화

| 항목 | 내용 |
|---|---|
| 결정 내용 | 기존 OpenAI API와 Ollama 연동 후보를 폐기하고 브라우저 Web Worker 기반 WebLLM을 생성형 표현 계층으로 사용한다. |
| 이유 | 백엔드와 API 키 없이 프론트엔드 단독 MVP, 개인정보 보호, 오프라인 가능성을 유지한다. |
| 체감 속도 원칙 | 규칙 엔진이 즉시 첫 반응을 표시하고, WebLLM은 백그라운드에서 짧은 후속 문장을 스트리밍한다. |
| 성능 예산 | 준비된 모델 기준 첫 토큰 800ms 이내 목표, 첫 화면 반응 100ms 이내, 일반 응답 1~3문장 및 출력 토큰 제한 |
| 복구 | 시간 예산 초과, WebGPU 미지원, 모델 미준비, 오류 시 규칙 응답만 유지 |
| 되돌릴 조건 | 목표 기기에서 WebLLM이 성능 예산과 품질 기준을 반복적으로 충족하지 못할 때 |

|---|---|
